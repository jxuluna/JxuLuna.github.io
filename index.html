面试准备之Vue

Posted on 2020-04-22 | In 面试准备

一. Vue和React的区别
1.监听数据变化的实现原理不同。

  • Vue通过数据劫持，能精确快速的计算出虚拟DOM的差异。在渲染的过程中，会跟踪每一个组件的依赖关系，不需要重新渲染整个组件树。
  • React默认是通过比较引用的方式进行的。

2.数据流向不同
3.模板渲染方式不同

  • React使用jsx渲染
  • Vue使用模板语法

二.Vue的nextTick的原理？
Vue是异步修改DOM并不鼓励直接接触DOM，使用nextTick后的回调。利用了事件循环的原理，把要执行的任务推到一个队列中，在下一次tick同步执行。

三.Vue的响应式系统

  • 任何一个Vue Component都有一个与之对应的Watcher实例
  • Vue的data上的属性会被添加getter和setter
  • 当Vue Component render函数被执行时，getter方法会被调用，此时Vue会记录此Vue Component所依赖的所有data。(此过程为依赖收集)

五.什么是MVVM？
Model：数据模型。
View：视图层。
ViewModel：Model和View之间的桥梁，处理具体的业务逻辑。
优点：代码解耦、提高可测试性、自动更新dom
缺点：bug难调试、数据大可维护性差。

六.Vue生命周期
其中created阶段组件实例已经创建，可以访问data、computed、watch、methods以及事件回调等，beforeMount阶段在挂载Dom之前被调用，调用相关的render函数，mounted阶段dom挂载完毕。
七.computed和watch的区别
computed：computed是计算属性，一般用于计算值，主要用于简化模板内的复杂运算。
computed有缓存性，在getter后会缓存，只有在依赖的属性值改变之后才会改变
watch：当监听后需要处理业务逻辑时用到
八.Proxy的优势
可以直接监听对象而非属性，可以直接监听数组的变化，有多种拦截方法。
proxy会返回新的对象，我们可以操作新的对象达到目的，而Object.defineProperty只能遍历对象的属性直接修改。

面试准备之前端性能优化

Posted on 2020-04-14 | In 面试准备

前端性能优化？

网络相关

1.代码压缩，减少HTTP请求
2.浏览器缓存：
强缓存：expires、cache-control
协商缓存：Last-Modified 和 If-Modified-Since、ETag 和 If-None-Match
3.DNS预解析

4.预加载： 会强制浏览器请求资源，不会阻塞onload事件，可以在一定程度上降低首屏加载的时间

优化渲染过程

1.懒执行
将某些逻辑延迟到使用后再计算，某些逻辑不需要在首屏就使用的，可以使用懒执行。一般可以使用定时器或事件调用来唤醒。
2.懒加载
懒加载就是将不关键的资源延后加载。
图片为例：只在定义区域内加载图片，先设置src属性为一张占位图，将真实的图片资源放入自定义属性中，
当进入定义区域内，将自定义属性替换为src属性，这样图片就会下载资源，实现懒加载。

文件优化

1.图片优化（减少像素点，减少每个像素点能显示的颜色）

  • 修饰类图片可以使用css代替
  • 小图片使用base64
  • 雪碧图
  • 选择正确的图片格式：小图使用png格式，图标可以用svg代替，照片用jpg

2.其他文件

  • css文件放在head
  • 服务器开启文件压缩功能
  • 将script放在body的底部，因为 JS 文件执行会阻塞渲染。可以给script加上defer，表示文件会并行下载，会在html解析完后顺序执行。async会在加载后立即执行。

静态资源使用CDN加载。

面试准备之JS

Posted on 2020-04-11 | In 面试准备

1.js的基本数据结构：

string，undefind，null，symbol，boolean，number

2.原型：

每个对象都有一个——proto——属性，指向原型，由他连接原型与对象，原型里又有prototype属性，该属性的constructor指回对象的构造函数形成一个闭环。

3.原型链：

每个对象都有proto属性，指向了创建了该对象的构造函数的原型，proto将对象连接起来形成原型链。用来实现继承和共享属性。

4.执行上下文

全局上下文，函数上下文，eval()上下文

5.闭包：

父函数在销毁的时候子函数仍然可以访问父函数中的变量对象，这样的函数称为闭包。

6.script引入方式：

  • html静态引入，动态引入

  • defer：在html加载后执行，按照顺序执行

  • async：异步加载，会阻塞渲染

    7.对象的拷贝

    （1）浅拷贝：

  • Object.assign(),

  • (…)展开运算符
    （2）深拷贝：

  • Json.parsr(Json.stringify(obj))
    缺点：当存在undefind或symbol时，无法拷贝，无法序列化函数，具有循环引用的对象时，会报错。

  • 递归拷贝

    8.new运算符的过程：

  • 创建一个新的对象

  • 链接到原型上：obj.__ proto __ = Con.prototype

  • 绑定this:apply(this)

  • 返回新的对象

    9.instanceof原理

    判断能否在原型链中找到该构造函数的prototype属性所指向的原型对象

    10.继承

    原型链继承，构造函数继承，组合继承

    11.类型转换

  • ‘-‘,’*’,’/‘ 一律转换为数字后计算

  • 数字 + 字符串 = 字符串， 运算顺序是从左到右
    数字 + 对象， 优先调用对象的valueOf -> toString
    数字 + boolean/null -> 数字
    数字 + undefined -> NaN

12.防抖与节流

  • 防抖 (debounce): 将多次高频操作优化为只在最后一次执行，通常使用的场景是：用户输入，只需再输入完成后做一次输入校验即可。
  • 节流：每隔一段时间执行一次，降低频率，将高频操作转换为低频操作。使用场景：滚动条事件。

    13.this的指向

    this指向与函数的运行环境有关（谁调用了，就指向谁）
    改变this的指向：
  • call: fn.call(target, 1, 2)
  • apply: fn.apply(target, [1,2])
  • bind: fn.bind(target, 1, 2)

    14.es6/es7

  • 声明
    • let/const：块级作用域，不存在变量提升，暂时性死区，不允许重复声明
    • const：定义常量，无法修改
      暂时性死区：声明的变量被绑定在该区域，不受外部影响。
  • 解构赋值
  • class/extends 类的声明与继承
  • Set
  • 异步：
    • promise
    • gengerator

      15.babel编译原理

  • babylon将es6/es7转换为AST（抽象语法树 (Abstract Syntax Tree)，是将代码逐字母解析成 树状对象 的形式）
  • babel-traverse 对 AST 进行遍历转译，得到新的 AST
  • 新 AST 通过 babel-generator 转换成 ES5

    16.数组api

面试准备之浏览器

Posted on 2020-04-04 | In 面试准备

什么是DOCTYPE及作用：

DOCTYPE是用来声明文档类型和DTD规范
HTML5

浏览器渲染过程

1.处理HTML并构建DOM树
2.处理CSS构建CSSOM树
3.将DOM与CSSOM树合并成一个渲染树
4.根据渲染树来布局，计算每个节点的位置
5.绘制并显示在屏幕上

重排reflow

定义： 布局或者几何属性需要改变
触发条件：

  • 增加、删除、修改DOM节点
  • 移动DOM的位置（动画）
  • 改变窗口大小的时候，或者滚动的时候
  • 修改网页的默认文字

    重绘repaint

    定义：当节点需要更改外观而不会影响布局
    触发条件：CSS改变

    跨标签通讯

  • 通过父页面window.open()和子页面postMessage
  • 同域下共享的localstoragr与监听window.onstorage
  • 设置cookie与不断轮询脏检查（setinterval）
  • 借助服务端或中间层

    跨域

  1. jsonp 通过script标签指向一个需要访问的地址并提供一个回调函数来接收数据（只限GET）
  2. cors 跨域资源共享 服务端设置Access-Control-Allow-Origin
  3. document.domain
    只适用于二级域名相同的情况下
    给页面添加 document.domain = ‘test.com’
  4. postMessage
    嵌入的第三方页面数据，一个页面发送消息，另一个页面接受。

垃圾回收机制

垃圾回收：将内存中不再使用的数据进行清理，释放出内存空间。

  • 新生代算法：用于存活较短的对象
  • 老生代算法：用于存活时间较长的
  • 对象

内存泄露

  • 意外的全局变量：无法被回收
  • 定时器：未被正确关闭，导致所引用的变量无法被释放
  • 事件监听没有被正确销毁
  • 闭包
  • dom引用

HTTP1.0和2.0的区别

相对于HTTP1.0，HTTP1.1的优化：

  • 1.0协议缺陷：

    • 无法复用链接，完成即断开，重新慢启动和TCP3次握手
    • 线头阻塞，导致请求之前互相影响

  • 1.1改进：

    • 长连接（keep-alive），复用
    • Host头处理
    • 新增断电续传、身份认证、错误通知的管理、增加一些cache缓存

  • 2.0：

    • 多路复用
    • 支持二进制传送
    • 头部压缩，减小传输的体积
    • 支持服务端推送

  • HTTPS：

    • 证书（公钥）
    • SSL加密
    • 端口443

安全

  • XSS攻击(跨站脚本攻击):注入恶意代码
    • cookie设置httpOnly
    • 转义页面上的输入内容和输出内容
  • CSRF(跨站请求伪造)
    • 不让第三方网站访问用户Cookie
    • 阻止第三方网站请求接口
    • 验证token
